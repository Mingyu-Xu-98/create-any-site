# Style Skills — 高品质前端设计技能包

## 触发条件（When to Activate）

当用户需要构建具有独特视觉风格的网站时激活。适用于需要避免"AI 模板感"、追求专业设计品质的场景。包括但不限于：配色优化、动效设计、排版调整、布局重构、视觉润色。

## 概述

Style Skills 是一套模块化的前端设计技能集，包含 23+ 个独立技能，覆盖从整体设计方向到细节打磨的完整设计流程。核心理念是**大胆的美学选择**（BOLD aesthetic choices），避免千篇一律的 AI 生成风格。

## 核心工作流

```
1. 收集设计上下文 → 2. 确定设计方向 → 3. 应用具体技能 → 4. 质量检查
```

### 第 0 步：设计上下文收集（强制）

在做任何设计之前，必须确认：
- 目标受众是谁？
- 使用场景是什么？
- 品牌个性和调性？
- 参考灵感或偏好？

### 第 1 步：选择设计方向

可选方向：极简主义、极繁主义、复古未来、有机自然、奢华质感、趣味玩乐、编辑风格、粗野主义、装饰艺术、柔软温暖、工业风格等。

## 技能清单

### 核心设计
| 技能 | 用途 | 何时使用 |
|------|------|---------|
| **frontend-design** | 核心设计指导，涵盖排版、配色、布局、动效、交互的完整方法论 | 每次设计任务必读 |
| **colorize** | 为单调的灰白界面引入战略性配色 | 界面太灰/缺乏视觉个性 |
| **typeset** | 优化字体选择、层级、字号、行距、可读性 | 排版感觉随意/不专业 |
| **arrange** | 改进布局、间距、视觉节奏 | 布局单调/间距不一致 |
| **animate** | 添加有目的的动效和微交互 | 界面太静态/缺少反馈 |

### 视觉调整
| 技能 | 用途 | 何时使用 |
|------|------|---------|
| **bolder** | 增强视觉冲击力和力量感 | 设计太温和/没有存在感 |
| **quieter** | 降低视觉噪音，增加留白和克制 | 设计太吵/元素过多 |
| **polish** | 整体视觉打磨，消除粗糙感 | 功能完成但视觉不够精致 |
| **delight** | 添加令人愉悦的微细节 | 设计正确但缺乏灵魂 |
| **overdrive** | 将某个设计方向推到极致 | 想要极端风格化的效果 |

### 分析与优化
| 技能 | 用途 | 何时使用 |
|------|------|---------|
| **audit** | 全面审查设计的一致性和质量 | 交付前质量检查 |
| **critique** | 提供建设性的设计批评 | 需要第二意见 |
| **optimize** | 性能优化，保持视觉效果 | 动效太多/加载慢 |
| **harden** | 增强边界情况和异常状态的处理 | 只做了 happy path |
| **clarify** | 简化复杂交互，提升信息架构清晰度 | 用户容易迷路/困惑 |

### 内容与呈现
| 技能 | 用途 | 何时使用 |
|------|------|---------|
| **distill** | 提炼冗余内容为简洁表达 | 文字太多/信息密度低 |
| **extract** | 从现有设计提取可复用的模式 | 需要建立设计系统 |
| **normalize** | 统一不一致的设计模式 | 多人协作导致风格不统一 |
| **adapt** | 适配不同平台/设备 | 需要响应式/跨端 |
| **onboard** | 设计新用户引导流程 | 产品有学习成本 |

## AI 模板感检测（Anti-Slop Test）

生成设计后自查：
- ❌ 是否使用了过度的毛玻璃效果？
- ❌ 是否用了 "Hero Section + 3 Card Grid" 标准布局？
- ❌ 是否所有圆角都一样大？
- ❌ 渐变是否千篇一律？
- ❌ 按钮是否都长一个样？
- ✅ 是否有让人一眼记住的独特视觉元素？

## 技能组合示例

**场景：从零设计一个作品集网站**
```
frontend-design（整体方向） → arrange（布局） → typeset（排版） → colorize（配色） → animate（动效） → polish（打磨）
```

**场景：优化已有设计**
```
audit（审查） → critique（批评） → 针对性使用 colorize/typeset/arrange → polish（最终打磨）
```
