# 故事讲述在用户体验设计中的应用

## 何时使用
- 设计目标包含传达用户情境、建立共情或解释复杂流程
- 需要统一界面布局、文案、交互反馈和视觉风格等设计元素
- 进行用户研究（如构建人物角色背景）、产品演示或团队协作对齐
- 产品需体现以人为本的设计理念，并融合跨学科视角（如戏剧、电影、工程）

## 如何执行
1. **明确核心叙事**：确定产品要传递的单一、清晰的用户体验目标或价值主张，确保所有设计决策服务于该故事。
2. **统一设计元素**：检查UI控件、文案、动效、色彩等组件是否协同强化同一核心信息，避免叙事冲突。
3. **借鉴戏剧与电影手法**：运用布景、灯光、节奏等叙事技巧，将界面视为“舞台”，使用户旅程具有起承转合。
4. **从真实经历提炼素材**：挖掘个人或用户的生活经历（如童年、职场、家庭场景），转化为具幽默感与深度的叙事片段。
5. **聚焦沟通目的**：确保故事真实、简洁且可被多元受众（工程师、设计师、普通用户）理解与接受。
6. **通过实践迭代优化**：组织工作坊、角色扮演或口头讲述练习，反复打磨故事表达，验证其共鸣力与清晰度。

## 注意事项
- 不适用于纯功能性、无用户交互的系统（如后台批处理工具）
- 不适用于无需情感或情境理解的技术文档（如API参考手册）
- 故事必须服务于设计目标，而非装饰性内容
