# 用户体验故事的叙事构建

## 何时使用
- 需要创建用户故事用于设计讨论、用户研究汇报或产品构思
- 目标是使故事更具共情力、揭示不同用户对同一情境的多元解读
- 需将用户塑造成“英雄”，产品作为“帮助者”解决其痛点
- 需通过多重视角展示文化、背景或价值观差异对体验的影响

## 如何执行
1. **整合五大核心要素**：
   - **视角（Perspective）**：决定由谁讲述，影响受众理解
   - **人物（Characters）**：使用真实或代表性用户角色
   - **背景（Context）**：提供时间、地点、环境等合理性支撑
   - **意象（Imagery）**：通过感官细节增强沉浸感
   - **语言（Language）**：选择适合受众的表达风格（口语化、技术性等）
2. **运用多重视角增强效果**：
   - 让主角亲自讲述，自然框定故事要点
   - 利用对比性视角（如HSBC广告）展示同一事物的不同解读
   - 根据视角差异推导设计需求（如夜晚=“黑暗不祥” vs “宁静平和”）
   - 创造意外情境形成认知反差（如本田广告中的地域文化对比）
3. **采用“英雄之旅”四阶段结构（可选）**：
   - **召唤冒险**：日常世界 → 问题出现 → 拒绝 → 获得帮助（产品/服务）
   - **启蒙与试炼**：遭遇障碍、挫折、同伴协助
   - **达成目标**：完成核心任务（如查到公交到站时间）
   - **回归**：安全返回日常世界，并分享所获（如告诉他人好消息）
4. **确保故事服务于UX目标**：
   - 用于激发共情、揭示痛点、引导设计讨论
   - 避免纯数据报告、技术说明书或无角色参与的内容
   - 始终从受众出发，明确其与故事的关系及你的讲述目的
