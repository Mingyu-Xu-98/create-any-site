# 面向管理者的故事策略

## 何时使用
- 听众为决策型管理者，关注“按时、按预算交付优质产品”
- 需在有限会议时间内快速获得支持或共识
- 需传达用户研究中的负面结果（如用户不喜欢某功能）
- 需启动新想法但避免陷入技术细节

## 如何执行
1. **识别管理者的核心目标与痛点**：聚焦其关心的“底线”——掌握全局、理解权衡、推动项目；避免讲述看似无关的故事。
2. **选择合适的故事使用时机**：
   - 启动新想法并简要说明背景
   - 传达用户研究中的负面发现
   - 同时呈现问题与解决方案
3. **构建有效故事结构**：
   - 开头强调已知价值（如“用户非常喜爱网站上的丰富信息”）
   - 引入具体人物案例（如“Mary在测试中反复绕圈三次”）
   - 提出具体改进建议（如“创建一个新横幅整合各主题信息”）
   - 配合粗糙但具象的草图，保留开放性以接受反馈
4. **避免常见错误**：
   - 切勿陷入详细规格或技术细节
   - 不要讲述冗长、缺乏明确要点的故事
   - 避免脱离管理者关心的产品目标
5. **增强说服力的关键技巧**：
   - 将新想法与管理者熟悉的场景关联
   - 强调故事中用户代表重要客户群体
   - 利用故事激发团队参与，使其更难轻易否定
6. **处理组织文化差异**：
   - 考虑听众的地理、职业或生活经验对问题感知的影响
   - 若涉及岗位变动担忧，用数据证明自动化仅处理常规咨询，复杂问题仍需人工
7. **与产品经理自有叙事协同**：思考你的故事如何补充或改变其用于激励销售、市场或客户的内部/外部故事。
