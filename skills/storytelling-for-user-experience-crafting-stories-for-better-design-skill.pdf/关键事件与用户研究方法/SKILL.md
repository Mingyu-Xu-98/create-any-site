# 关键事件与用户研究方法

## 何时使用
- 研究目标是深入理解用户在特定情境下的行为、情感或系统影响
- 需要访问真实用户或利益相关者
- 需要设定安全、信任的访谈环境
- 不适用于需要量化数据或探索抽象偏好的场景

## 执行步骤
1. **引导具体经历**：
   - 不询问总体看法，而是引导讲述“具体的近期经历”
   - 使用开放式问题：“请告诉我一次某个系统帮助或损害了您工作的经历”
2. **鼓励详细描述**：
   - 背景、过程、结果及个人感受
   - 特别关注“关键时刻”——对任务成败、情绪或安全产生重大影响的瞬间
3. **记录上下文细节**：
   - 物理环境、人员角色、时间压力、系统状态
   - 包括物理布局、工具摆放、着装、照明、协作模式等
4. **分析转化洞察**：
   - 识别深层需求、未被满足的痛点、系统可用性缺陷
   - 特别注意“可记忆性”问题：长时间稳定运行后恢复界面的设计
5. **保留情感强度**：
   - 在转化故事时保留原始叙述中的情感强度和后果严重性
   - 强化团队对可用性重要性的认知

## 输出标准
输出为一组带有上下文的用户故事，可用于识别设计机会、验证假设或提升团队对用户体验后果严重性的共情，特别适用于医疗信息系统等高风险领域。
