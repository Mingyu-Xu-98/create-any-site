# 可用性测试中的情境故事整合

## 何时使用
- 可用性测试任务依赖用户工作流程、环境或外部工具
- 用户行为受上下文强烈影响（如在冷藏库操作设备）
- 需将分散的用户行为片段整合为系统性问题叙事
- 需向团队同时传递界面问题与深层使用情境

## 如何执行
1. **测试前收集“即时故事”**：
   - 请参与者描述最近使用系统的实际场景
   - 包括搜索目的、工作环境、遇到的挑战、使用的其他工具
2. **定制个性化测试任务**：
   - 基于其真实经历设计任务，确保相关性和现实意义
   - 例如：“上次您提到需要查找某类文献，请尝试完成类似搜索”
3. **观察并记录两类信息**：
   - **交互层面**：用户如何操作界面（如展开/折叠字段）
   - **情境层面**：提及的工作流程、信息需求、外部资源使用
4. **识别并整合“有吸引力的故事片段”**：
   - 提取用户引述、行为描述、情绪反应
   - 横向比对多个参与者，寻找重复模式
   - 构建复合叙事，揭示单一观察无法体现的系统性问题
5. **生成两份结构化报告**：
   - **可用性报告**：聚焦界面设计问题，提供具体改进建议
   - **情境洞察报告**：整理用户故事、工作模式、使用上下文，用于产品战略
6. **可视化叙事**：
   - 使用截图序列（故事板）按时间顺序展示用户路径
   - 标注关键决策点和困惑时刻
   - 仅描述问题，不提出新设计方案
