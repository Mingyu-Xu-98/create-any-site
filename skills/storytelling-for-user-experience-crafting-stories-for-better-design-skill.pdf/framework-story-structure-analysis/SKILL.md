# 分析框架故事结构

## 何时使用
- 用户提交的叙事文本以高度相似的场景、状态或角色处境开始和结束
- 结尾虽形式上“回到原点”，但主角（或叙述者）对事物的理解、态度或认知已发生变化
- 需要判断该文本是否构成有效的框架故事，而非“故事中的故事”结构

## 执行步骤

1. **确认是否为框架故事**：
   - 检查开头与结尾是否发生在相同或高度相似的情境中（如地点、状态、角色处境）
   - 确认结尾虽回归原点，但主角的认知、态度或理解已发生可识别的变化
   - 验证故事整体由碎片化片段或意象组成，单独看不构成完整故事，仅组合后形成整体意义

2. **排除“故事中的故事”结构**：
   - 若文本包含多个可独立存在的子故事，彼此交织并相互阐明，则不属于框架故事

3. **识别框架结构变体**：
   - **Me-Them-Me**：起始于“我”的特质或观点 → 转向“他们”的不同做法或视角 → 回归“我”并体现新理解
   - **Here-There-Here**：从某地出发 → 前往他处经历事件 → 返回原地但心理/认知已变
   - **Now-Then-Now**：从当前时刻切入 → 回溯过去经历 → 回到现在并赋予当前状态新意义

4. **输出结果**：
   - 明确指出识别出的框架结构类型
   - 描述主角在结尾处相较于开头所获得的核心认知转变
   - 注明叙述视角（第一人称或第三人称）

> 注意：即使素材看似简单（如日常轶事），只要满足上述结构特征，即视为有效框架故事。
