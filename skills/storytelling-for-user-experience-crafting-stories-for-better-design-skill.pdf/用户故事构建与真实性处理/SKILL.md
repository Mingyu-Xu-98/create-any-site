# 用户故事构建与真实性处理

## 何时使用
- 需要传达用户情境、激发设计灵感或建立团队共情
- 团队成员质疑故事在数据沟通中的科学性
- 原始用户叙述包含可能被目标受众误解为能力不足的语言表达
- 需要将真实用户经历转化为设计洞察

## 执行步骤
1. **确保故事基本要素**：
   - 包含事件序列（行动、决策、反应）
   - 明确动机和目标（可显性或隐性）
   - 置于文化或情境背景中
2. **验证真实性基础**：
   - 基于真实用户访谈、观察或调研数据
   - 与科研论文同样有效，非虚构编造
3. **进行伦理性编辑**：
   - 识别可能被误读的语言表达（非标准语法、方言等）
   - 保留原始术语和句式结构，仅修正明显语法错误
   - 避免过度标准化导致“精英化”
4. **利用文化背景**：允许听众基于既有经验填补未明说细节，但需引导正确解读。
5. **建立主动沟通机制**：使故事不仅能传递信息，还能在人与人之间建立连接。

## 输出标准
生成一个具备解释力、情境性和共情潜力的用户故事，既忠实于用户原始体验，又能在目标语境中维护其尊严，可用于传达用户需求、触发设计思路或促进团队共识。
