# 用户体验与叙事设计技能集

## 触发条件（When to Activate）

当网站需要叙事性内容时激活：品牌故事页、用户案例展示、"关于我们"、产品介绍、用户证言、个人经历展示。也适用于需要引导用户情感共鸣、跨团队对齐产品愿景的场景。

## 在网站构建中的应用

| 页面类型 | 推荐技能组合 |
|---------|-------------|
| 品牌故事页 | ux-storytelling-framework → 叙事结构与要素设计 → written-story-crafting-for-ux |
| 用户案例展示 | 用户研究中的故事收集 → 用户故事构建与真实性处理 → visual-narrative-enhancement |
| 产品介绍 | 面向管理者的故事策略 → 多模态演示设计 → story-pacing-management |
| 个人作品集 | 用户体验故事的叙事构建 → 叙事结构与要素设计 → visual-narrative-enhancement |
| 无障碍倡导 | story-based-accessibility-advocacy → 伦理与真实性准则 |

## 叙事五要素（核心）

每段网站内容应包含：**背景**（场景）→ **角色**（真实用户）→ **目标**（想达成什么）→ **冲突**（障碍/痛点）→ **解决**（产品如何帮助）

---

本技能集聚焦于如何在用户体验（UX）设计、用户研究、产品沟通与跨职能协作中有效运用故事讲述（Storytelling）方法。通过这些技能，可构建真实、有说服力且符合伦理的用户故事。

## How to Use
当您需要在用户体验、产品设计或团队协作中运用故事方法时，请根据任务目标、受众类型和当前阶段，参考上方分类或表格选择最匹配的技能。调用对应技能时，提供具体上下文（如用户数据、听众角色、沟通目标），AI 将基于该技能的指导原则为您提供定制化建议或输出。
