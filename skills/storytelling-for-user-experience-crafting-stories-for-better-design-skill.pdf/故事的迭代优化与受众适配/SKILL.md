# 故事的迭代优化与受众适配

## 何时使用
- 故事用于正式演示、设计评审或跨团队沟通
- 目标听众为技术专家、工程师等注重细节的群体
- 初次讲述后受众反应不佳（如困惑、注意力分散）
- 需将用户研究转化为可操作的设计输入

## 如何执行
1. **撰写初稿并首次讲述**：
   - 包含角色、动机、问题解决逻辑
   - 在真实场景中向目标受众讲述
2. **观察反应并收集反馈**：
   - 注意笑声、困惑、注意力分散等非语言信号
   - 记录直接反馈（如“太长”、“不适用”、“缺少技术细节”）
3. **迭代修改**：
   - 调整长度（如从多段落精简为两句话）
   - 增删情节细节（如加入技术维度）
   - 重组叙事流程或融合旧故事片段创造新故事
4. **针对技术受众特别优化**：
   - 使用贴近现状的代表性角色和情境
   - 用硬数据支持故事选择
   - 确保行动具体可感知（如字段大小、标签措辞）
   - 准确使用技术术语，证明“理解各方语言”
   - 关联可立即解决的问题或高度吸引兴趣的挑战
5. **评估是否继续使用**：
   - 若多次尝试仍无效，判断是否：
     - 受众不匹配（好故事用于错误受众）
     - 目标不适合用故事表达
   - 此时应“知道何时放手”，保留故事待合适时机
6. **理解故事在UX中的核心作用**：
   - 将设计锚定于真实情境
   - 确保“人”始终处于设计中心
   - 帮助收集共享用户信息、激发创意、建立共同目标感
   - 通过真实用户故事（如开放大学案例）将枯燥信息转化为情感连接
