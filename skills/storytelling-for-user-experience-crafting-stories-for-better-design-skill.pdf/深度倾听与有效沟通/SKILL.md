# 深度倾听与有效沟通

## 何时使用
- 进行用户访谈、设计会议、销售拜访或客户沟通
- 需要理解话语背后的隐含需求、情绪或未明说问题
- 团队沟通变成“一系列独白”而非真正对话
- 需要捕捉“潜台词”和“弦外之音”

## 执行步骤
1. **识别倾听障碍**：
   - 等待发言机会而非专注倾听
   - 分心行为（查看邮件、内心独白等）
   - 伪倾听（表面在听，实则准备回应）
2. **实施深度倾听**：
   - 全神贯注理解说话者意图，而非等待插话
   - 给予对方充分表达时间，包括沉默时刻
   - 理解话语背后的“冰山之下”部分（情绪、需求、未明说问题）
3. **创造安全环境**：
   - 确保每位参与者都被认真倾听
   - 让每个人能自由表达而不受打断威胁
   - 作为引导者监督“倾听优先”规则执行
4. **应用倾听成果**：
   - 将倾听所得融入后续沟通和设计方案
   - 在提案中编织对客户问题的认知
   - 促进更多有意义的故事被讲述

## 输出标准
通过深度倾听准确理解真实需求，调整设计方案、提升提案说服力，并建立基于信任的协作关系，体现为更贴合业务与用户需求的产出及积极的利益相关者反馈。
