# 用户体验书面故事创作

## 何时使用
- 需通过书面形式向未知或分散受众传达用户洞察
- 用于人物角色、用户研究报告、设计文档等UX交付物
- 不适用于需要即时反馈的口头交流场景

## 如何执行
1. **控制长度与精炼度**：
   - 保持“所需长度即可，不多不少”
   - 高效设定上下文，仅保留关键细节
   - 避免冗余描写或旁白评论

2. **选择叙述视角**：
   - 第一人称：以核心角色身份讲述
   - 第三人称：外部叙述者描述事件

3. **直接呈现用户信息**：
   - “直接讲故事”，而非“讲一个关于故事的故事”
   - 将结论内嵌于故事本身，或通过引言/注释明确点出

4. **运用信息设计工具**：
   - 利用留白、插图等视觉元素塑造页面呈现与叙事节奏
   - 通过排版（如文本框）突出故事

5. **文化与社会暗示**：
   - 使用共享“常识”传递隐含信息（如“玩Skatepunkz游戏到12级”暗示学生身份）
   - 确保读者能从相同线索提取一致含义

6. **质量保障**：
   - 完稿前请他人阅读（作者会脑补语调，读者仅依赖文字）
   - 主动识别并解构故事所依赖的文化/社会预设
