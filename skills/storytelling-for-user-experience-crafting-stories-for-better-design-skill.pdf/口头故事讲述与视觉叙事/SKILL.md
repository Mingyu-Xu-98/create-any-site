# 口头故事讲述与视觉叙事

## 何时使用
- 内容需通过口头方式向现场听众传达
- 需要向多元背景团队（IT、管理层、跨语言成员）清晰传达用户行为或设计概念
- 首次讲述某故事或向无法看见的听众（网络研讨会、电话）讲述
- 需要建立深层人际连接而非仅传递信息

## 执行步骤
1. **口头讲述准备**：
   - 多次阅读故事，分析吸引力和情绪氛围
   - 研究背景文化含义，准备回答问题
   - 将角色场景内化，确保关键事实自然融入
   - 在脑中可视化感官细节（声音、气味、色彩等）
2. **发展个人风格**：
   - 基于真诚、开放、清晰和诚实
   - 避免表演腔，接近自然说话方式
   - 利用多模态元素（语调、表情、手势、眼神）
3. **应用视觉叙事**：
   - 选择合适形式：漫画（浓缩复杂问题）或故事板（展示完整交互流程）
   - 克服绘图技能不足：使用火柴人、简单线条和在线工具
   - 展示非UI细节：时序、位置、尺寸、颜色、情绪、空间关系
4. **实时调整互动**：
   - 根据听众反应调整节奏、语气
   - 利用口头讲述的同步反馈循环
   - 通过提问或邀请参与深度连接听众

## 输出标准
完成准备流程后，能自信、自然且有效地向目标听众传达故事，建立情感连接；或生成包含上下文、时序、用户行为及情绪的视觉叙事作品，用于团队沟通和设计共识。
