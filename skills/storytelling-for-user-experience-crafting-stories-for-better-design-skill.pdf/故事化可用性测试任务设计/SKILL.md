# 故事化可用性测试任务设计

## 何时使用
- 可用性测试任务显得枯燥或缺乏情境
- 参与者来自倾向于避免直接批评设计的文化背景（如亚洲文化）
- 用户研究者缺乏目标用户的专业领域知识（如医疗领域）
- 需要参与者更投入并自然指出界面问题

## 执行步骤
1. **评估文化背景**：识别目标用户的沟通偏好，特别是是否倾向于回避负面反馈。
2. **构建情境故事**：创建包含以下元素的情感丰富故事：
   - 明确的角色关系
   - 戏剧性冲突
   - 具体任务目标
   - 时间紧迫感
3. **邀请用户共创**（适用于专业领域）：
   - 以通用语言启动任务故事
   - 邀请专业用户基于真实经验完善故事细节
   - 将用户自创情境作为测试任务基础
4. **替代干巴巴指令**：用故事化任务替代传统的操作说明。
5. **调整复杂度**：根据实际需求调整故事复杂度，但确保包含基本叙事元素（角色、动机、冲突、目标）。

## 输出标准
生成一个具有叙事张力的可用性测试任务脚本，使参与者更投入并自然暴露使用障碍，从而获得更真实的用户反馈。
