# 用户研究中的故事收集与倾听

## 何时使用
- 参与者仅提供简短、无细节的回答（如“我买了一件毛衣”）
- 直接追问无效，对话陷入沉默或停滞
- 参与者提及他人、个人习惯或使用情绪化语言
- 需在真实环境中（如家庭、办公室）唤起对现实因素的记忆

## 如何执行
1. **预留探索性时间**：在研究议程中专门留出非结构化对话时间
2. **识别故事触发信号**：
   - 提及家人/同事的行为对比
   - 描述个人仪式或决策策略
   - 使用具体场景或情绪化语言
3. **使用开放式引导语**：
   - “告诉我关于那件事的情况。”
   - “你上次做这件事是什么时候？能详细说说吗？”
4. **应用最小干预对话技巧**：
   - **回声法**：重复最后几个词作为疑问（如“……让人困惑？”）
   - **对话失衡法**：以升调结尾促使对方补充（如“然后您……？”）
5. **分享个人小故事（当直接提问无效时）**：
   - 准备简短、真实的个人经历（几句话内）
   - 确保故事真实，不可虚构
   - 分享后提出开放式问题（如“你买这件毛衣是给谁的？”）
6. **营造心理安全空间**：
   - 不打断叙述
   - 使用点头、眼神接触等非语言信号
   - 在焦点小组中利用“故事连锁效应”鼓励他人分享
7. **多源收集故事**：
   - 直接观察：在真实环境中记录言语、行为、情绪
   - 间接挖掘：分析日志、客服记录、社交媒体、销售反馈
   - 整合显性与隐性故事，构建全面用户体验视图
