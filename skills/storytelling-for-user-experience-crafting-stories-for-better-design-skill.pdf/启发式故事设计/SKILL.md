# 启发式故事设计

## 何时使用
- 需要激发听众对问题的新思考方式并促使其自主构想解决方案
- 目标是启动关于产品/交互/体验的设计讨论
- 团队需要突破既有思维框架以生成新颖的用户体验场景
- 需要避免直接提供方案，而是引导听众“拥有”想法

## 执行步骤
1. **选择故事类型**：
   - **弹簧板故事**：描述熟悉困境，暗示未来变化但不提供具体方案
   - **讨论启动故事**：聚焦当前痛点，以开放式问题结尾
   - **故事接龙**：使用People/Places/Activities/Motivations词汇表组合生成新场景
2. **构建核心要素**：
   - 弹簧板故事：包含典型困境 + 未来暗示
   - 讨论启动故事：包含足够细节 + 想象空间
   - 故事接龙：确保包含人物、地点、动机、活动
3. **保留开放性**：
   - 不提供完整解决方案
   - 在关键环节留白，促使听众主动填补
   - 以开放式问题结尾
4. **鼓励非常规组合**（故事接龙）：
   - 使用看似夸张的元素组合
   - 纳入低频用户类型
   - 强调宽泛类别而非精细差异

## 输出标准
生成一个能有效引导听众聚焦特定问题并提出创新方案的启发式故事，使听众认为解决方案是“自己的主意”，从而更愿意采取行动。
