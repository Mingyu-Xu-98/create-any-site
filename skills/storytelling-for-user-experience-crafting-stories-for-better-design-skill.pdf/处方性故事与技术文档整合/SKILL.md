# 处方性故事与技术文档整合

## 何时使用
- 设计进入详细规范阶段
- 需要在技术细节讨论中保持用户真实使用情境可见
- 团队包含非技术角色（如临床人员与IT人员）
- 需要防止遗忘功能初衷或使用方式

## 执行步骤
1. **确认适用场景**：
   - 设计涉及复杂交互或多模式渠道
   - 需为不同类型用户描述差异化体验
   - 技术标准制定者与终端用户存在理解鸿沟
2. **构建技术规范故事**：包含五大组成元素：
   - **前提假设**：支撑故事体验的基本假设
   - **体验片段**：一两句话构成的具体用户场景
   - **目标**：期望达成的新用户体验
   - **参考文献**：引用受众尊重的权威来源
   - **要点摘要**：可视化或高度凝练的关键信息
3. **整合到设计流程**：
   - 用于生成新设计想法
   - 文档化设计决策
   - 评估设计方案
   - 促进跨职能团队理解
4. **选择输出形式**：
   - 结合故事板格式
   - 明确标注前提条件、叙事流程和后置条件

## 输出标准
生成一份结构化的处方性故事文档，包含前提、体验、目标、参考和要点，用于指导技术规范编写并促进跨职能团队对齐，确保设计方案始终以用户为中心。
