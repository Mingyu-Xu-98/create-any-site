# 用户体验故事的伦理与真实性准则

## 何时使用
- 复述用户访谈、可用性测试或观察中收集的真实经历
- 故事包含情感冲击或挑战团队既有信念的内容
- 需向专家或决策者呈现用户洞察
- 构建复合原型或人物角色配套故事

## 如何执行
1. **遵循三大伦理原则**：
   - **诚实**：准确呈现用户研究内容，不得歪曲证据
   - **真实**：忠实于原始事件的情感氛围，以参与者可能讲述的方式叙述
   - **简洁清晰**：仅使用足够传达清晰信息的细节
2. **确保专业伦理合规**：
   - 研究前获得参与者知情同意，明确信息用途
   - 确认匿名性偏好，保护隐私与尊严
   - 主动建立对所有相关方有益的合作关系
   - 确保工作不会损害参与者安全、尊严或隐私
3. **维护故事真实性**：
   - 允许为匿名修改细节（如姓名、地点），但不得扭曲核心事实
   - 不选择仅因有趣但属异常值（outlier）的故事
   - 不回避传递不利信息的故事（如挑战现有设计方向）
4. **识别并管理“反思性”影响**：
   - 警惕自身存在对参与者的影响（如建议术语、引导性提问）
   - 避免将用户认知框架转换为研究者自身框架
5. **针对专家受众特别注意**：
   - 保持贴近原始素材，失实细节会分散其注意力
   - 必要时可迅速回溯至详细原始记录
6. **区分现实主义故事特征**：
   - 采用第三人称叙述
   - 作者隐匿，不插入主观评论
   - 聚焦可观察的行为、对话或明确思想
   - 使用具体引用（如“刷掉站牌上的雪”、“输入短信查询”）
   - 暗示事件“真实发生或可能发生”
